# Microservices Demo Project

This repository provides a sample application built with a microservices architecture. It is intended to demonstrate the core concepts and practical implementation of microservices in a simple, easy-to-understand project.

The application includes the following components:

* **API Gateway** – Serves as the single entry point for client requests.
* **Config Server** – Centralizes configuration management for all services.
* **Discovery Server** – Handles service registration and discovery.
* **Student Service** – Manages student-related operations.
* **School Service** – Manages school-related operations.

This project is ideal for developers who want to explore the fundamentals of microservices, including service discovery, centralized configuration, API gateway routing, and inter-service communication.

## Project Structure

```text
├── api-gateway/
├── config-server/
├── discovery-server/
├── student-service/
└── school-service/
```

## Features

* Microservices architecture
* API Gateway for request routing
* Centralized configuration
* Service registration and discovery
* Independent Student and School services
* Clean and modular project structure

## Purpose

The primary goal of this repository is to serve as a learning resource for understanding how microservices work together in a distributed system. It can also be used as a starting point for building more advanced microservices-based applications.
